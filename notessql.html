Notes on SQL

These notes provide a rough overview of SQL. I do not claim complete accuracy, but they appear to be correct.

Type the following in the terminal and enter the password: 

mysql -u root -p

The following query asks the server to return its version number and the current date: 

mysql> select version(), current_date;
+-----------+--------------+
| version() | current_date |
+-----------+--------------+
| 9.4.0     | 2025-08-24   |
+-----------+--------------+
1 row in set (0.006 sec)

The above query is simple, but it reveals several things about MySQL:

  • A query ends with a semicolon. There are some exceptions.
  • When you issue a query, MySQL sends it to the server for execution and displays the results.
  • MySQL displays the results in tabular form. Normally, column labels are the names of the columns you fetch, but if you are retrieving the value of an expression, MySQL labels the column using the expression itself.
  • MySQL shows how many rows were returned and how long the query took to execute. Be aware that this only gives you a rough idea of performance, as execution time can depend on several other factors.

Keywords may be entered in any letter case. The following query demonstrates that you can use MySQL as a simple calculator: 

mysql> select sin(pi()/4), (4+1)*5;
+--------------------+---------+
| sin(pi()/4)        | (4+1)*5 |
+--------------------+---------+
| 0.7071067811865475 |      25 |
+--------------------+---------+
1 row in set (0.018 sec)

You can enter multiple statements on a single line; they need to be separated by a semicolon. A query can also span multiple lines. MySQL determines where your statement ends by looking for the terminating semicolon. If you decide you do not want to execute a query while you're in the process of entering it, cancel it by typing \c: 

mysql> select 
    -> user()
    -> \c
mysql>

Use the SHOW statement to find out what databases currently exist on the server: 

mysql> show databases;
+--------------------+
| Database           |
+--------------------+
| information_schema |
| mysql              |
| performance_schema |
| sys                |
+--------------------+
4 rows in set (0.201 sec)

If there isn't a database for you to use, you can create it yourself: 

create database test;

Depending on the system, database names (and also table names) are case-sensitive, so it is best to use the same lettercase. Creating a database does not select it for use; you must do that explicitly: 

use test;

Your database needs to be created only once, but you must select it for use each time you begin a mysql session. The database you just created is empty, as the following query shows: 

mysql> show tables;
Empty set (0.002 sec)

The simplest way to display columns from a table is to use SELECT * FROM table_name. The * specifier is a shortcut that means all columns: 

mysql> select * from Orders;
+---------+------------+------------+------------+-----------+
| OrderID | CustomerID | EmployeeID | OrderDate  | ShipperID |
+---------+------------+------------+------------+-----------+
|   10248 |         90 |          5 | 1996-07-04 |         3 |
|   10249 |         81 |          6 | 1996-07-05 |         1 |
|   10250 |         34 |          4 | 1996-07-08 |         2 |
...

Naming columns explicitly enables you to select only the ones of interest, in any order: 

mysql> select CustomerID, OrderDate from Orders;
+------------+------------+
| CustomerID | OrderDate  |
+------------+------------+
|         90 | 1996-07-04 |
|         81 | 1996-07-05 |
|         34 | 1996-07-08 |
...

Unless you qualify or restrict a SELECT query in some way, it retrieves every row in your table. You can provide a WHERE clause that specifies one or more conditions that rows must satisfy: 

mysql> select CustomerName, City from Customers where City = "Berlin";
+---------------------+--------+
| CustomerName        | City   |
+---------------------+--------+
| Alfreds Futterkiste | Berlin |
+---------------------+--------+
1 row in set (0.002 sec)

Conditions can test for equality, inequality or relative ordering. For some types of data, such as strings, you can use pattern matches. The following statement selects only those customers that live in a city that begin with the letter 'b': 

mysql> select CustomerName, City from Customers where City LIKE  "B%";
+------------------------------+--------------+
| CustomerName                 | City         |
+------------------------------+--------------+
| Alfreds Futterkiste          | Berlin       |
| Cactus Comidas para llevar   | Buenos Aires |
| Chop-suey Chinese            | Bern         |
| Folk och fä HB               | Bräcke       |
...

The LIKE operator in the previous query performs a pattern match, where % acts as a wildcard that matches any string. A WHERE clause can test multiple conditions and different conditions can test different columns: 

mysql> select * from Customers where City like "B%" and CustomerName like "A%";
+------------+---------------------+--------------+---------------+--------+------------+---------+
| CustomerID | CustomerName        | ContactName  | Address       | City   | PostalCode | Country |
+------------+---------------------+--------------+---------------+--------+------------+---------+
|          1 | Alfreds Futterkiste | Maria Anders | Obere Str. 57 | Berlin | 12209      | Germany |
+------------+---------------------+--------------+---------------+--------+------------+---------+
1 row in set (0.001 sec)

Output columns can be calculated by evaluating expressions. This query combines two different columns using CONCAT(): 

mysql> select concat(CustomerID, ', ', CustomerName) from Customers;
+------------------------------------------+
| concat(CustomerID, ', ', CustomerName)   |
+------------------------------------------+
| 1, Alfreds Futterkiste                   |
| 2, Ana Trujillo Emparedados y helados    |
...

Inside a table, a column often contains many duplicate values; and sometimes you only want to list the distinct values. This can be done using the SELECT DISTINCT statement: 

mysql> select distinct Country from Customers;
+-------------+
| Country     |
+-------------+
| Germany     |
| Mexico      |
| UK          |
...

SQL requires single quotes around text values (most database systems will also allow double quotes). However, numeric fields not be enclosed in quotes: 

mysql> select * from Customers where CustomerID = 1;
+------------+---------------------+--------------+---------------+--------+------------+---------+
| CustomerID | CustomerName        | ContactName  | Address       | City   | PostalCode | Country |
+------------+---------------------+--------------+---------------+--------+------------+---------+
|          1 | Alfreds Futterkiste | Maria Anders | Obere Str. 57 | Berlin | 12209      | Germany |
+------------+---------------------+--------------+---------------+--------+------------+---------+
1 row in set (0.004 sec)

The following operators can be used in the WHERE clause:
= (equal), <> or != (not equal), > (greater than), < (less than), >= (greater than or equal to), <= (less than or equal to), BETWEEN (within a range), LIKE (pattern matching), and IN (match any value in a list).

The WHERE clause can be combined with the usual logical operators AND, OR, and NOT operators: 

mysql> select * from Customers where Country = 'Germany' and (City = 'Berlin' or City = 'Stuttgart'); 
+------------+---------------------+--------------+-------------------+-----------+------------+---------+
| CustomerID | CustomerName        | ContactName  | Address           | City      | PostalCode | Country |
+------------+---------------------+--------------+-------------------+-----------+------------+---------+
|          1 | Alfreds Futterkiste | Maria Anders | Obere Str. 57     | Berlin    | 12209      | Germany |
|         86 | Die Wandernde Kuh   | Rita Müller  | Adenauerallee 900 | Stuttgart | 70563      | Germany |
+------------+---------------------+--------------+-------------------+-----------+------------+---------+
2 rows in set (0.005 sec)

mysql> select * from Customers where not Country = 'Germany' and not Country = 'USA';
+------------+--------------------------------------+----------------------+-------------------------------------------------+-----------------+------------+-------------+
| CustomerID | CustomerName                         | ContactName          | Address                                         | City            | PostalCode | Country     |
+------------+--------------------------------------+----------------------+-------------------------------------------------+-----------------+------------+-------------+
|          2 | Ana Trujillo Emparedados y helados   | Ana Trujillo         | Avda. de la Constitución 2222                   | México D.F.     | 05021      | Mexico      |
...

The ORDER BY keyword is used to sort the result-set in ascending or descending order. It sorts in ascending order by default. To sort the results in descending order, use the DESC keyword. 

mysql> select * from Customers order by Country desc;
+------------+--------------------------------------+----------------------+-------------------------------------------------+-----------------+------------+-------------+
| CustomerID | CustomerName                         | ContactName          | Address                                         | City            | PostalCode | Country     |
+------------+--------------------------------------+----------------------+-------------------------------------------------+-----------------+------------+-------------+
|         35 | HILARIÓN-Abastos                     | Carlos Hernández     | Carrera 22 con Ave. Carlos Soublette #8-35      | San Cristóbal   | 5022       | Venezuela   |
...

It is also possible to order by several columns. If two rows have the same values in the first column, they will be ordered by the next column, and so on. 

mysql> select * from Customers order by Country asc, CustomerName desc;
+------------+--------------------------------------+----------------------+-------------------------------------------------+-----------------+------------+-------------+
| CustomerID | CustomerName                         | ContactName          | Address                                         | City            | PostalCode | Country     |
+------------+--------------------------------------+----------------------+-------------------------------------------------+-----------------+------------+-------------+
|         64 | Rancho grande                        | Sergio Gutiérrez     | Av. del Libertador 900                          | Buenos Aires    | 1010       | Argentina   |
...
